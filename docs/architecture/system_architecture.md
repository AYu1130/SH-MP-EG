# 系统架构设计

## 1.总体架构

本系统采用 **感知层 — 边缘层 — 网络层 — 应用层** 的四层架构，边缘智能网关处于核心位置

┌─────────────────────────────────────────────────────────────┐
│                      应用层 (Application)                    │
│  ┌─────────────────┐  ┌─────────────────────────────────┐  │

│  └─────────────────┘  └─────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ MQTT / HTTP
┌─────────────────────────────────────────────────────────────┐
│                      网络层 (Network)                        |
│  ┌───────────────────────────────────────────────────────┐  │
│  │              MQTT Broker (EMQX)                       │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ 统一 JSON
┌─────────────────────────────────────────────────────────────┐
│                    边缘层 (Edge Gateway)                    │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────┐   │

## 2.模块划分

### 2.1 感知层

- ESP32-S3节点：采集环境数据（温湿度、光照），通过Wi-Fi发送至网关
- STM32F103+HM-10：采集数据，通过蓝牙BLE广播或连接发送至网关

### 2.2 边缘层（树莓派网关）

- Wi-Fi接收模块：基于Flask提供REST API，接收HTTP POST数据
- BLE接收模块：基于bleak库扫描并连接BLE设备，接收数据
- 协议转换模块：将不同协议的数据统一转换成JSON格式，device_id、protocol、payload、timestamp 等字段
- MQTT发布模块：是同paho-mqtt将统一格式的数据发布到EMQX Broker

### 2.3 网络层

- MQTT Broker：使用EMQX（Doker部署），负责消息路由和转发

### 2.4 应用层

- 上位机：通过MQTT订阅数据，可使用Web页面或Node-RED Dashboard展示
- Node-RED：实现本地智能联动规则（如温度超限报警，光照自动控灯）

## 3.数据流

1. 感知层节点采集数据，通过WiFi（HTTP）或BLE发送到网关
2. 边缘层接收数据，解析并转换为统一JSON
3. 网关将统一JSON发布到EMQX Broker的指定主题（gateway/devices）
4. 应用层订阅主题，接收数据并展示；Node-RED订阅主题，根据规则触发动作

## 4.关键设计决策

- 统一数据类型：采用JSON Schema规范，确保不同来源的数据具有一致结构
- 模块化适配器：每种协议独立实现，便于扩展新协议
- 边缘自治：本地Node-RED规则引擎，可独立运行，不依赖于云服务
